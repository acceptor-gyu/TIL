# 새로운 TIL 항목 생성 및 작성

사용자 입력: $ARGUMENTS

## 1단계: 파일 생성

다음 규칙에 따라 파일을 생성하세요.

### 날짜 및 번호 결정

```bash
date +%y%m%d
```

같은 날짜의 기존 파일 수를 확인하여 번호를 결정합니다 (01, 02, 03...).

### 카테고리 및 파일명 결정

`$ARGUMENTS`를 다음과 같이 파싱하세요:
- **첫 번째 단어**: 카테고리 (예: AI, Database, Network)
- **나머지 단어들**: 제목

예시:
- `AI Claude Code 사용법` → 카테고리: AI, 제목: Claude Code 사용법
- `Database Redis 캐싱 전략` → 카테고리: Database, 제목: Redis 캐싱 전략

파일명 규칙: `YYMMDD_번호_제목_언더스코어.md`
저장 위치: `카테고리/파일명.md`

카테고리 폴더가 없으면 새로 생성합니다.

### 파일 템플릿

```markdown
# [제목]

## 개요
[학습한 내용의 간단한 소개]

## 상세 내용

### 1. [소주제]

### 2. [소주제]

### 3. [소주제]

## 핵심 정리
- 핵심 포인트 1
- 핵심 포인트 2
- 핵심 포인트 3

## 키워드

### [키워드1]
[설명]

### [키워드2]
[설명]

## 참고 자료
-
```

## 2단계: 내용 작성

파일을 생성한 후, **검색 및 조사를 통해 실제 내용을 작성**하세요.

- 제목과 관련된 핵심 개념, 동작 원리, 실용적인 예제를 포함합니다
- 상세 내용은 3~8개의 소주제로 구성합니다
- 코드 예제가 있으면 반드시 포함합니다
- 키워드는 5~10개 선정하고 각각 설명을 작성합니다
- 핵심 정리는 이 TIL에서 가장 중요한 3~5가지를 bullet point로 작성합니다

## 3단계: README 업데이트

`README.md`의 해당 카테고리 섹션 **최상단**에 다음 형식으로 추가하세요:

```
- [제목](./카테고리/파일명.md) - YYYY.MM.DD
```

## 완료 시 출력

작업 완료 후 다음을 출력하세요:
- 생성된 파일 경로
- 작성된 주요 소주제 목록
