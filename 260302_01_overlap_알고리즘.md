# Overlap 알고리즘

## 개요
두 구간(interval)이 겹치는지 판별하고, 겹치는 영역을 계산하는 알고리즘에 대해 학습한다.
구간 겹침 문제는 회의실 배정, 구간 병합, 스케줄 충돌 감지 등 실무와 코딩 인터뷰에서 자주 등장하는 패턴이다.

## 구간 겹침 판별 조건

두 구간 `[A_start, A_end]`와 `[B_start, B_end]`가 있을 때, **겹치지 않는 조건**은 다음 두 가지다.

```
B_end < A_start  (B가 A보다 완전히 왼쪽에 있음)
A_end < B_start  (A가 B보다 완전히 왼쪽에 있음)
```

De Morgan 법칙을 적용하면, **겹치는 조건(닫힌 구간)**은 아래와 같다.

```
A_start <= B_end && B_start <= A_end
```

**닫힌 구간 vs 열린 구간 차이:**
- 닫힌 구간 `[A_start, A_end]`: 끝점이 같아도 겹침 (`A_start <= B_end && B_start <= A_end`)
- 열린 구간 `(A_start, A_end)`: 끝점만 닿는 경우는 겹치지 않음 (`A_start < B_end && B_start < A_end`)

예를 들어 `[1, 3]`과 `[3, 5]`는 닫힌 구간에서는 겹치지만, 열린 구간에서는 겹치지 않는다.

## 1차원 구간 겹침 (Interval Overlap)

### 겹침 판별

```java
public boolean isOverlap(int aStart, int aEnd, int bStart, int bEnd) {
    // 겹치는 조건: 두 구간이 서로 완전히 분리되지 않은 경우
    return aStart <= bEnd && bStart <= aEnd;
}
```

### 겹치는 영역 계산

두 구간이 겹친다면, 겹치는 구간은 `[max(A_start, B_start), min(A_end, B_end)]` 이다.

```java
public int[] overlapRange(int aStart, int aEnd, int bStart, int bEnd) {
    if (!isOverlap(aStart, aEnd, bStart, bEnd)) {
        return null; // 겹치지 않음
    }
    int overlapStart = Math.max(aStart, bStart);
    int overlapEnd = Math.min(aEnd, bEnd);
    return new int[]{overlapStart, overlapEnd};
}
```

### 겹치는 길이 계산

```java
public int overlapLength(int aStart, int aEnd, int bStart, int bEnd) {
    int overlapStart = Math.max(aStart, bStart);
    int overlapEnd = Math.min(aEnd, bEnd);
    // 겹치는 길이가 음수이면 겹치지 않으므로 0 반환
    return Math.max(0, overlapEnd - overlapStart);
}
```

**예시:**
- `[1, 5]`와 `[3, 8]` → 겹치는 구간: `[3, 5]`, 길이: `2`
- `[1, 3]`과 `[5, 8]` → 겹치지 않음, 길이: `0`

## 2차원 영역 겹침 (Rectangle Overlap)

두 직사각형의 겹침은 **x축과 y축 각각에 1차원 겹침 조건을 동시에 만족**해야 한다.

두 직사각형 A: `(x1, y1) ~ (x2, y2)`, B: `(x3, y3) ~ (x4, y4)` 가 있을 때:

```
겹침 조건: x축 겹침 AND y축 겹침
  x축 겹침: x1 <= x4 && x3 <= x2
  y축 겹침: y1 <= y4 && y3 <= y2
```

### 겹침 판별 코드

```java
public boolean isRectangleOverlap(int x1, int y1, int x2, int y2,
                                   int x3, int y3, int x4, int y4) {
    boolean xOverlap = x1 <= x4 && x3 <= x2;
    boolean yOverlap = y1 <= y4 && y3 <= y2;
    return xOverlap && yOverlap;
}
```

### 겹치는 넓이 계산

```java
public int overlapArea(int x1, int y1, int x2, int y2,
                        int x3, int y3, int x4, int y4) {
    int overlapWidth = Math.max(0, Math.min(x2, x4) - Math.max(x1, x3));
    int overlapHeight = Math.max(0, Math.min(y2, y4) - Math.max(y1, y3));
    return overlapWidth * overlapHeight;
}
```

**예시:**
- A: `(0, 0) ~ (4, 4)`, B: `(2, 2) ~ (6, 6)`
  - x 겹침: `[2, 4]` → 너비 2
  - y 겹침: `[2, 4]` → 높이 2
  - 겹치는 넓이: `2 * 2 = 4`

## Sweep Line과 Overlap

Sweep Line(스윕 라인)은 구간을 **시작점과 끝점 이벤트로 분리**하여 정렬 후 순회하는 기법이다.
모든 구간 쌍을 비교하는 O(N²) 대신, O(N log N)으로 겹침 문제를 해결할 수 있다.

### 핵심 아이디어

1. 각 구간을 `(위치, 이벤트_종류)` 형태로 변환
2. 시작 이벤트: `+1`, 종료 이벤트: `-1`
3. 위치 기준 정렬 후 순회하며 카운터 증감
4. 카운터의 최대값 = 한 지점에서 겹치는 구간의 최대 개수

### 최대 겹침 개수 구하기

```java
public int maxOverlap(int[][] intervals) {
    // 이벤트 배열: [위치, 종류(+1: 시작, -1: 종료)]
    int[][] events = new int[intervals.length * 2][2];
    int idx = 0;
    for (int[] interval : intervals) {
        events[idx++] = new int[]{interval[0], 1};   // 시작 이벤트
        events[idx++] = new int[]{interval[1], -1};  // 종료 이벤트
    }

    // 위치 기준 정렬, 같은 위치면 종료(-1) 먼저 처리 (닫힌 구간이면 시작 먼저)
    Arrays.sort(events, (a, b) -> a[0] != b[0] ? a[0] - b[0] : a[1] - b[1]);

    int maxCount = 0;
    int currentCount = 0;
    for (int[] event : events) {
        currentCount += event[1];
        maxCount = Math.max(maxCount, currentCount);
    }
    return maxCount;
}
```

**예시:**
- 구간: `[1,4], [2,5], [3,6]`
- 이벤트: `(1,+1), (2,+1), (3,+1), (4,-1), (5,-1), (6,-1)`
- 최대 겹침: 3 (위치 3~4 사이에 세 구간 모두 겹침)

## 활용 사례

### 회의실 배정 문제 (Meeting Rooms)

"주어진 회의 일정이 겹치지 않으면 한 사람이 모든 회의에 참석 가능한가?"
→ 구간을 시작 시간으로 정렬 후, 인접한 구간끼리 겹침 여부만 확인

```java
public boolean canAttendMeetings(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    for (int i = 1; i < intervals.length; i++) {
        // 이전 회의 종료 전에 다음 회의 시작하면 겹침
        if (intervals[i][0] < intervals[i - 1][1]) {
            return false;
        }
    }
    return true;
}
```

### 구간 병합 (Merge Intervals)

겹치는 구간들을 하나로 합쳐 겹치지 않는 최소 구간 집합 반환:

```java
public int[][] mergeIntervals(int[][] intervals) {
    // 시작점 기준 정렬
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

    List<int[]> merged = new ArrayList<>();
    merged.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] last = merged.get(merged.size() - 1);
        int[] curr = intervals[i];

        if (curr[0] <= last[1]) {
            // 겹치면 끝점을 더 큰 값으로 확장
            last[1] = Math.max(last[1], curr[1]);
        } else {
            // 겹치지 않으면 새 구간 추가
            merged.add(curr);
        }
    }
    return merged.toArray(new int[0][]);
}
```

### 구간 삽입 (Insert Interval)

정렬된 구간 배열에 새 구간을 삽입하고 겹치는 구간을 병합:

```java
public int[][] insertInterval(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0;
    int n = intervals.length;

    // 새 구간보다 앞에 있는 구간 그대로 추가
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // 겹치는 구간 모두 병합
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // 새 구간보다 뒤에 있는 구간 그대로 추가
    while (i < n) {
        result.add(intervals[i++]);
    }
    return result.toArray(new int[0][]);
}
```

## 시간 복잡도 분석

| 연산 | 시간 복잡도 | 설명 |
|------|------------|------|
| 단일 판별 | O(1) | 두 구간의 시작/끝점만 비교 |
| N개 구간 병합 | O(N log N) | 정렬 O(N log N) + 순회 O(N) |
| 최대 겹침 개수 (Sweep Line) | O(N log N) | 이벤트 생성 O(N) + 정렬 O(N log N) |
| 구간 삽입 (정렬된 배열) | O(N) | 한 번의 순회로 처리 가능 |

## 핵심 정리
- 두 구간의 **겹치지 않는 조건의 역(부정)**으로 겹침 조건을 도출한다: `A_start <= B_end && B_start <= A_end`
- 2차원 직사각형 겹침은 x축, y축 **각각의 1차원 겹침을 동시에 만족**해야 한다
- N개 구간 문제는 **시작 시간 기준 정렬** 후 인접 비교로 O(N log N) 해결이 가능하다
- **Sweep Line**은 구간을 이벤트(시작/종료)로 분리하여 최대 겹침 개수 등 복잡한 문제를 O(N log N)에 처리한다
- 구간 병합 시 끝점은 항상 `max(last.end, curr.end)`로 갱신해야 한다 (한 구간이 다른 구간 안에 완전히 포함되는 경우 대비)

## 키워드
`Overlap`: 두 구간 또는 영역이 서로 겹치는 상태

`Interval`: 시작점과 끝점으로 정의되는 1차원 구간 `[start, end]`

`구간 겹침`: 두 구간이 공통으로 속하는 구간이 존재하는지 판별하는 것

`Sweep Line`: 가상의 선을 한 방향으로 이동시키며 이벤트를 순서대로 처리하는 기법

`Merge Intervals`: 겹치는 모든 구간을 하나로 합쳐 최소 개수의 비겹침 구간을 반환하는 알고리즘

`Rectangle Overlap`: 두 직사각형의 겹침 여부를 x축·y축 각각의 1차원 겹침으로 판별하는 기법

`구간 병합`: 겹치거나 인접한 구간을 하나의 큰 구간으로 합치는 연산

`시간 복잡도`: 알고리즘이 입력 크기 N에 따라 수행되는 연산 횟수의 증가율 (구간 문제는 대부분 O(N log N))
