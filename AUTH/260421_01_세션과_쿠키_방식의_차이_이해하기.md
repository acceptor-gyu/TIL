# 세션과 쿠키 방식의 차이 이해하기

## 개요
웹 애플리케이션에서 사용자 인증과 상태 유지를 위해 사용되는 세션(Session)과 쿠키(Cookie)의 동작 원리, 차이점, 그리고 각각의 적합한 사용 시나리오를 정리한다.

## 상세 내용

### 1. 쿠키(Cookie)란 무엇인가
- 쿠키의 정의와 동작 원리
- 클라이언트 측 저장 방식의 특징
- 쿠키의 구성 요소 (Name, Value, Domain, Path, Expires, HttpOnly, Secure, SameSite)
- 쿠키의 종류: Session Cookie vs Persistent Cookie

### 2. 세션(Session)이란 무엇인가
- 세션의 정의와 동작 원리
- 서버 측 저장 방식의 특징
- Session ID 발급과 전달 과정
- 세션 저장소의 종류 (In-Memory, File, Database, Redis)

### 3. 쿠키 기반 인증 vs 세션 기반 인증
- 쿠키만으로 인증 정보를 관리하는 방식의 장단점
- 세션 + 쿠키 조합으로 인증하는 방식의 장단점
- 보안 관점에서의 비교 (탈취, 위변조, XSS, CSRF)
- 확장성(Scalability) 관점에서의 비교

### 4. 다중 서버 환경에서의 세션 관리 문제
- Sticky Session
- Session Clustering
- 외부 세션 저장소 (Redis, Memcached)

### 5. 세션/쿠키 vs 토큰 기반 인증(JWT)과의 비교
- Stateful vs Stateless 인증의 차이
- 각 방식의 트레이드오프
- 실무에서의 선택 기준
