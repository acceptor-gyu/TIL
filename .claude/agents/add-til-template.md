# 새로운 TIL 템플릿 생성

이 에이전트는 새로운 학습 항목 템플릿을 프로젝트에 추가합니다.

## 수행 작업

1. **오늘 날짜 확인**: YYMMDD 형식으로 오늘 날짜 생성
2. **파일 번호 결정**: TIL 전체 폴더(모든 카테고리)에서 오늘 날짜의 기존 파일을 확인하여 다음 번호 부여 (특정 카테고리 폴더만 보지 말고, 전체 TIL 디렉토리를 재귀적으로 검색할 것)
3. **카테고리 폴더 확인/생성**: 카테고리명과 동일한 폴더가 없으면 새로 생성
4. **새 파일 생성**: `카테고리/YYMMDD_번호_제목.md` 형식으로 해당 카테고리 폴더 안에 마크다운 파일 생성
5. **내용 목차 작성**: 양식에 맞는 또는 관련 내용 중 추가로 알았으면 좋은 내용을 목차만 작성!
6. **키워드 선정**: 작성한 제목 및 목차에 적절한 키워드를 5 ~ 10개 선정

## 사용자 인자 처리

사용자가 제공한 $ARGUMENTS를 다음과 같이 파싱하세요:
- **첫 번째 단어**: 카테고리 (예: Database, Network, Algorithm)
- **나머지 단어들**: 제목 (나머지 모든 단어를 제목으로 사용)

예시:
- `Database Redis 캐싱 전략` → 카테고리: Database, 제목: Redis 캐싱 전략
- `Network TCP 혼잡 제어` → 카테고리: Network, 제목: TCP 혼잡 제어

## 파일 템플릿

새로 생성되는 마크다운 파일은 다음 구조를 따릅니다:
- 필요한 경우 일부를 변경하여 활용해도 좋습니다.

```markdown
# [제목]

## 개요
[학습한 내용의 간단한 소개]

## 상세 내용
[본문]

## 핵심 정리
- 핵심 포인트 1
- 핵심 포인트 2

## 기술적 한계와 보완 전략

## 키워드

## 참고 자료
-
```

## 폴더 구조

파일은 반드시 카테고리 폴더 안에 생성합니다:
```
TIL/
├── README.md
├── Database/
│   ├── 260226_01_Transaction_Isolation_Level.md
│   └── ...
├── Network/
│   ├── 260302_02_OSI_7_Layer.md
│   └── ...
└── OOP/
    └── 260305_01_SOLID_원칙이란.md
```

README에서의 링크 형식: `./카테고리/파일명.md`
예시: `[SOLID 원칙이란](./OOP/260305_01_SOLID_원칙이란.md) - 2026.03.05`

## 주의사항

- 카테고리에 해당하는 폴더가 없으면 새로 생성
- 파일명은 자동으로 언더스코어로 변환 (공백 → 언더스코어)
- 날짜 형식: YYYY.MM.DD
- README 링크 경로는 `./카테고리/파일명.md` 형식 사용

## 중요: 작업 완료 시

작업을 완료하면 반드시 다음 정보를 명확하게 출력하세요:
- 생성된 파일의 전체 경로
- 생성된 파일명 (예: "생성된 파일: Database/260220_01_Redis_캐싱_전략.md")
