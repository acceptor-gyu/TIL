# 새로운 TIL 항목 추가

이 skill은 새로운 학습 항목을 프로젝트에 추가합니다.

## 수행 작업

1. **오늘 날짜 확인**: YYMMDD 형식으로 오늘 날짜 생성
2. **파일 번호 결정**: 같은 날짜의 기존 파일을 확인하여 다음 번호 부여
3. **카테고리 폴더 확인/생성**: 카테고리명과 동일한 폴더가 없으면 새로 생성
4. **새 파일 생성**: `카테고리/YYMMDD_번호_제목.md` 형식으로 해당 카테고리 폴더 안에 마크다운 파일 생성
5. **내용 목차 작성**: 양식에 맞는 또는 관련 내용 중 추가로 알았으면 좋은 내용을 목차만 작성!
6. **키워드 선정**: 작성한 제목 및 목차에 적절한 키워드를 5 ~ 10개 선정

## 사용법

```
/add-til-template [카테고리] [제목]
```

**예시:**
- `/add-til-template Database Redis 캐싱 전략`
- `/add-til-template Network TCP 혼잡 제어`
- `/add-til-template Algorithm 이진 탐색 트리`

## 인자 처리

- **$ARGUMENTS[0]** 또는 **$0**: 카테고리 (예: Database, Network, Algorithm)
- **$ARGUMENTS[1..]** 또는 **$1 이후**: 제목 (나머지 모든 단어를 제목으로 사용)

## 파일 템플릿

새로 생성되는 마크다운 파일은 다음 구조를 따릅니다:
- 필요한 경우 일부를 변경하여 활용해도 좋습니다.

```markdown
# [제목]

## 개요
[학습한 내용의 간단한 소개]

## 상세 내용
[본문]

## 핵심 정리
- 핵심 포인트 1
- 핵심 포인트 2

## 키워드

## 참고 자료
-
```

## 폴더 구조

파일은 반드시 카테고리 폴더 안에 생성합니다:
```
TIL/
├── README.md
├── Database/
│   ├── 260226_01_Transaction_Isolation_Level.md
│   └── ...
├── Network/
│   ├── 260302_02_OSI_7_Layer.md
│   └── ...
└── OOP/
    └── 260305_01_SOLID_원칙이란.md
```

README에서의 링크 형식: `./카테고리/파일명.md`
예시: `[SOLID 원칙이란](./OOP/260305_01_SOLID_원칙이란.md) - 2026.03.05`

## 주의사항

- 카테고리에 해당하는 폴더가 없으면 새로 생성
- 카테고리가 README에 없으면 새 섹션 추가
- 파일명은 자동으로 스네이크 케이스로 변환 (공백 → 언더스코어)
- 날짜 형식: YYYY.MM.DD
- README 링크 경로는 `./카테고리/파일명.md` 형식 사용

## 실행 후 확인

1. 카테고리 폴더 안에 새 마크다운 파일이 생성되었는지 확인
2. README.md 링크 경로가 `./카테고리/파일명.md` 형식인지 확인
3. 사용자에게 생성된 파일 경로와 다음 단계 안내
