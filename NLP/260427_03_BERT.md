# BERT (Bidirectional Encoder Representations from Transformers)

## 개요
Google이 2018년에 발표한 사전 학습 언어 모델로, Transformer의 Encoder 구조를 기반으로 양방향 문맥을 동시에 학습하여 자연어 이해(NLU) 태스크에서 혁신적인 성능 향상을 이끌어낸 모델이다. Jacob Devlin, Ming-Wei Chang, Kenton Lee, Kristina Toutanova가 공동 저자이며, 논문 발표 당시 GLUE를 포함한 11개 NLP 태스크에서 최고 성능(SOTA)을 기록했다.

## 상세 내용

### 1. BERT의 핵심 아이디어 - 양방향(Bidirectional) 학습

**기존 언어 모델과의 차이**

기존 언어 모델은 크게 두 가지 방식으로 나뉜다.

- **단방향(Unidirectional)**: GPT처럼 왼쪽에서 오른쪽으로만 문맥을 읽는 방식. "나는 ___을 먹었다" 에서 빈칸 이전 단어만 참조한다.
- **얕은 양방향(Shallow Bidirectional)**: ELMo처럼 순방향과 역방향 LSTM을 별도로 학습한 뒤 결합하는 방식. 두 방향이 서로 독립적이라 진정한 양방향 문맥 이해가 어렵다.

BERT는 문장 전체의 모든 토큰이 서로를 참조할 수 있는 **깊은 양방향(Deep Bidirectional)** 구조를 채택했다.

**왜 양방향이 중요한가**

"나는 은행에 갔다" 라는 문장에서 '은행'이 금융 기관인지 강가(river bank)인지를 파악하려면 문장 전체 문맥이 필요하다. 단방향 모델은 '은행' 이후 단어를 볼 수 없으므로 이런 중의적 표현을 정확히 처리하기 어렵다. BERT의 양방향 Attention은 모든 토큰이 문장 내 나머지 모든 토큰을 동시에 참조하여 풍부한 문맥 표현을 만든다.

---

### 2. BERT의 아키텍처

**Transformer Encoder 구조**

BERT는 Transformer에서 Encoder 블록만 사용한다. 각 Encoder 블록은 다음으로 구성된다.

- Multi-Head Self-Attention: 입력 시퀀스의 모든 토큰 쌍 간 Attention 스코어를 계산
- Feed-Forward Network: 각 토큰 위치에 독립적으로 적용되는 2층 완전 연결 레이어
- Layer Normalization + Residual Connection

**BERT-Base vs BERT-Large 비교**

| 항목 | BERT-Base | BERT-Large |
|---|---|---|
| Encoder 레이어 수 (L) | 12 | 24 |
| 히든 사이즈 (H) | 768 | 1024 |
| Attention 헤드 수 (A) | 12 | 16 |
| 파라미터 수 | 110M | 340M |

**Input Representation**

BERT의 입력은 세 가지 임베딩의 합산으로 구성된다.

```
Input = Token Embedding + Segment Embedding + Position Embedding
```

- **Token Embedding**: WordPiece로 분리된 각 서브워드 토큰의 임베딩
- **Segment Embedding**: 문장 A에 속하는지, 문장 B에 속하는지 구분하는 0/1 임베딩
- **Position Embedding**: 각 토큰의 순서 정보를 담는 임베딩 (학습 가능한 파라미터)

---

### 3. 사전 학습 (Pre-training) 전략

BERT는 두 가지 태스크를 동시에 학습하며 사전 학습된다.

**Masked Language Model (MLM)**

일반적인 언어 모델처럼 다음 단어를 예측하면 양방향 학습이 불가능하다. BERT는 이 문제를 해결하기 위해 입력 토큰의 15%를 랜덤으로 선택한 뒤 다음 비율로 변형한다.

- 80%: `[MASK]` 토큰으로 교체 → 모델이 원래 단어를 예측하도록 학습
- 10%: 랜덤한 다른 단어로 교체 → 모델이 모든 토큰을 주의 깊게 보도록 강제
- 10%: 변경 없이 원래 단어 유지 → 실제 단어 표현도 학습

이 비율 조합은 파인튜닝 시 `[MASK]` 토큰이 존재하지 않는 불일치(Pre-training vs Fine-tuning 불일치) 문제를 완화하기 위한 설계다.

**Next Sentence Prediction (NSP)**

QA나 NLI(자연어 추론)처럼 두 문장의 관계를 이해하는 태스크를 지원하기 위해 도입된 사전 학습 목표다.

- 50%: 실제로 연속된 문장 쌍 (IsNext 레이블)
- 50%: 무작위로 선택된 문장 쌍 (NotNext 레이블)

입력 형식: `[CLS] 문장A [SEP] 문장B [SEP]`

모델은 `[CLS]` 토큰의 최종 표현으로 IsNext/NotNext를 이진 분류한다.

**학습 데이터**

- BooksCorpus (800M 단어)
- English Wikipedia (2,500M 단어)

---

### 4. 파인튜닝 (Fine-tuning) 방법론

사전 학습된 BERT 위에 태스크별 레이어를 추가하고, BERT 전체 파라미터를 함께 미세 조정한다.

**분류(Classification)**

`[CLS]` 토큰의 최종 히든 벡터(768차원)를 분류 레이어에 통과시킨다. 감성 분석, 스팸 분류 등에 활용된다.

```
[CLS] 입력 문장 [SEP]
  └─ [CLS] 벡터 → Linear Layer → Softmax → 클래스 예측
```

**질의응답(QA) - SQuAD**

질문과 지문을 함께 입력하고, 각 토큰이 정답의 시작(start) 또는 끝(end) 위치인지 예측한다.

```
[CLS] 질문 [SEP] 지문 [SEP]
  └─ 각 토큰 벡터 → Start/End 위치 예측
```

**개체명 인식(NER)**

토큰 단위 분류 태스크로, 각 토큰의 최종 벡터를 개별적으로 분류 레이어에 통과시킨다.

```
[CLS] 토큰1 토큰2 ... [SEP]
  └─ 각 토큰 벡터 → 개체명 태그(B-PER, I-ORG 등) 분류
```

**문장 유사도**

두 문장을 `[SEP]`로 구분하여 입력하고, `[CLS]` 벡터로 유사도를 예측한다.

### 5. Tokenizer - WordPiece

**WordPiece 동작 원리**

WordPiece는 단어를 서브워드(Subword) 단위로 분리하는 알고리즘이다. 어휘 사전에 존재하는 가장 긴 서브워드 단위를 찾아 분리하며, 단어 내부 서브워드에는 `##` 접두사를 붙여 원래 단어의 연속임을 표시한다.

```
"playing" → ["play", "##ing"]
"tokenization" → ["token", "##ization"]
"unhappiness" → ["un", "##happiness"] 또는 ["un", "##happi", "##ness"]
```

**OOV(Out-of-Vocabulary) 문제 해결**

학습 데이터에 없는 단어도 서브워드로 분리하면 어휘 사전 내 조각들로 표현이 가능하다. 예를 들어 신조어 "코로나19"도 "코", "##로", "##나", "##19" 같은 서브워드 조합으로 처리할 수 있다.

**특수 토큰**

| 토큰 | 역할 |
|---|---|
| `[CLS]` | 모든 입력 시퀀스의 맨 앞에 위치. 시퀀스 전체를 대표하는 집약 벡터로 사용 |
| `[SEP]` | 문장 경계를 구분. 단일 문장 끝 또는 두 문장 사이에 삽입 |
| `[MASK]` | MLM 사전 학습 시 마스킹된 토큰 자리를 표시 |
| `[PAD]` | 배치 내 시퀀스 길이를 맞추기 위한 패딩 토큰 |
| `[UNK]` | 어휘 사전에 없는 토큰을 대체 |

### 6. BERT의 한계와 후속 모델

**BERT의 한계**

- **높은 사전 학습 비용**: BERT-Large 학습에 TPU 64개로 4일 소요
- **최대 시퀀스 길이 512**: 긴 문서 처리가 구조적으로 제한됨
- **[MASK] 토큰 불일치**: 파인튜닝 단계에서는 `[MASK]` 토큰이 없어 사전 학습과 파인튜닝 간 도메인 차이가 생김
- **NSP의 실효성 논란**: 너무 쉬운 태스크여서 실질적 도움이 제한적
- **생성 태스크에 취약**: Decoder 구조가 없어 텍스트 생성에 직접 활용 불가

**후속 모델 비교**

| 모델 | 핵심 개선 사항 |
|---|---|
| **RoBERTa** (2019, Facebook) | NSP 제거, 더 많은 데이터와 긴 학습, 동적 마스킹 적용 |
| **ALBERT** (2019, Google) | 파라미터 공유와 임베딩 분해로 18배 경량화, SOP(문장 순서 예측) 도입 |
| **DistilBERT** (2019, HuggingFace) | 지식 증류(Knowledge Distillation)로 BERT 크기 40% 축소, 성능 97% 유지 |
| **ELECTRA** (2020, Google) | 대체 토큰 탐지(RTD)로 전체 토큰을 학습에 활용, 동일 연산 예산 대비 높은 효율 |

ELECTRA는 Generator가 일부 토큰을 그럴듯한 단어로 교체하고, Discriminator가 각 토큰이 원본인지 교체된 것인지 판별하는 방식으로 학습한다. MLM의 15%만 학습하는 비효율을 극복하여 소형 모델에서 특히 강력하다.

### 7. 한국어 BERT 모델

**KoBERT (SKT Brain)**

- SKT Brain이 공개한 한국어 특화 BERT 모델
- 파라미터: 약 9200만 개
- 학습 데이터: 한국어 Wikipedia + 뉴스 데이터 (약 3.2억 단어)
- 토크나이저: **SentencePiece** (어휘 사전 크기 8,002개)
- WordPiece 대신 SentencePiece를 사용하여 공백 없이 한국어 서브워드를 처리

```python
# KoBERT SentencePiece 토크나이저 예시
"한국어 모델을 공유합니다"
→ ['▁한국', '어', '▁모델', '을', '▁공유', '합니다', '.']
```

**ETRI KoBERT**

- 한국전자통신연구원이 개발한 모델
- 형태소 분석기 기반 토크나이저 사용 (어휘 사전 30,797개)
- 약 47억 형태소로 학습
- 형태소 단위 토크나이징이 한국어의 교착어 특성에 더 적합하다는 장점

**한국어 토크나이저 선택 시 고려사항**

- 한국어는 교착어(조사, 어미가 결합)로 WordPiece 분리 결과가 의미 단위와 일치하지 않을 수 있음
- 형태소 분석기(Mecab, Komoran 등) 전처리 후 BERT에 입력하면 토큰 효율이 향상되나 추론 지연이 증가
- 도메인 특화(의료, 법률, 특허 등)의 경우 해당 데이터로 추가 사전 학습(Domain-Adaptive Pre-training)을 수행하면 성능이 크게 향상됨

## 핵심 정리

- BERT는 Transformer Encoder를 쌓아 MLM과 NSP로 양방향 문맥을 사전 학습한 모델이다.
- MLM에서 전체 토큰의 15%를 마스킹하고, 마스킹된 토큰의 80%를 `[MASK]`로 교체, 10%는 랜덤 단어, 10%는 원본 유지하는 비율 설계가 핵심이다.
- `[CLS]` 토큰은 시퀀스 전체를 집약하는 벡터로 분류 태스크에 활용되고, `[SEP]`는 문장 경계를 구분한다.
- WordPiece 서브워드 토크나이징으로 OOV 문제를 해결하며, 한국어에서는 SentencePiece나 형태소 분석기 기반 토크나이저가 주로 사용된다.
- RoBERTa는 NSP 제거, ALBERT는 경량화, DistilBERT는 지식 증류, ELECTRA는 대체 토큰 탐지로 각각 BERT의 한계를 개선했다.

## 키워드

`BERT` - Bidirectional Encoder Representations from Transformers. Google이 2018년 발표한 사전 학습 언어 모델로, Transformer Encoder를 기반으로 양방향 문맥을 학습한다.

`Transformer Encoder` - 입력 시퀀스 전체를 Self-Attention으로 처리하는 블록. BERT는 이 Encoder를 12개(Base) 또는 24개(Large) 쌓아 구성한다.

`Masked Language Model` - MLM. 입력 토큰의 15%를 마스킹하고 원래 단어를 예측하게 하는 사전 학습 목표. 단방향의 한계 없이 양방향 문맥 학습을 가능하게 한다.

`Next Sentence Prediction` - NSP. 두 문장이 실제로 연속된 문장인지 판별하는 사전 학습 태스크. 문장 간 관계 이해를 위해 도입되었으나, RoBERTa에서 불필요함이 밝혀져 제거되었다.

`Fine-tuning` - 사전 학습된 BERT에 태스크별 레이어를 추가하고 전체 파라미터를 미세 조정하는 과정. 분류, QA, NER 등 다양한 다운스트림 태스크에 적용된다.

`WordPiece Tokenizer` - 단어를 서브워드 단위로 분리하는 알고리즘. 어휘 사전에 없는 단어(OOV)도 서브워드 조합으로 처리하며, 단어 내부 서브워드에는 `##` 접두사를 사용한다.

`양방향 문맥 학습` - 입력 시퀀스의 모든 토큰이 왼쪽과 오른쪽 문맥을 동시에 참조하는 학습 방식. BERT의 가장 큰 혁신으로, Self-Attention 구조와 MLM이 이를 가능하게 한다.

`사전 학습 언어 모델` - Pre-trained Language Model. 대규모 비지도 데이터로 먼저 언어의 일반적 표현을 학습하고, 이후 소규모 지도 데이터로 파인튜닝하는 Transfer Learning 패러다임.

`Transfer Learning` - 사전 학습된 모델의 파라미터를 새로운 태스크에 재사용하는 기법. BERT에서는 BooksCorpus와 Wikipedia로 학습한 표현을 각 NLP 태스크에 전이한다.

`Attention Mechanism` - 쿼리(Query), 키(Key), 값(Value)의 내적으로 토큰 간 가중치를 계산하는 메커니즘. BERT의 Multi-Head Self-Attention은 여러 Attention 헤드가 병렬로 서로 다른 문맥 관계를 포착한다.
