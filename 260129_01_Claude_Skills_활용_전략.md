# Claude Skills 활용 전략

## 개요
Claude Code에서 제공하는 Skills 시스템을 효과적으로 활용하는 방법과 전략을 학습합니다. Skills는 반복적인 작업을 자동화하고, 프로젝트별 워크플로우를 커스터마이징할 수 있는 강력한 기능입니다.

## 상세 내용

### Skills란?
- Claude Code에서 실행 가능한 커스텀 명령어 시스템
- `.claude/skills/` 디렉토리에 정의
- 슬래시 커맨드 형태로 실행 (예: `/add-til`, `/commit`, `/review-pr`)
- 프롬프트 기반으로 동작하며, 재사용 가능한 워크플로우 제공

### Skills의 구성 요소

#### 1. skill.md (필수)
Skills의 핵심 프롬프트 파일:
- 작업 목적과 수행할 단계 명시
- 입력 파라미터 처리 방법 정의
- 출력 형식 및 사용자 피드백 방식 지정

#### 2. skill.json (필수)
Skills의 메타데이터:
```json
{
  "name": "skill-name",
  "description": "간단한 설명",
  "version": "1.0.0"
}
```

### Skills 설계 전략

#### 1. 단일 책임 원칙
- 하나의 Skill은 하나의 명확한 작업만 수행
- 복잡한 워크플로우는 여러 Skills로 분리

#### 2. 명확한 인터페이스
- 파라미터는 직관적이고 간단하게
- 사용 예시를 skill.md에 명시
- 에러 케이스 처리 방법 정의

#### 3. 프로젝트 컨텍스트 활용
- 프로젝트 구조와 컨벤션 반영
- 기존 파일과의 일관성 유지
- README 등 문서 자동 업데이트

### 실전 활용 사례

#### TIL 관리 자동화
```
/add-til [카테고리] [제목]
```
- 날짜 기반 파일명 자동 생성
- README 자동 업데이트
- 일관된 템플릿 적용

#### Git 워크플로우 개선
```
/commit
```
- 스테이징된 변경사항 분석
- 컨벤션에 맞는 커밋 메시지 자동 생성
- Co-authored-by 자동 추가

#### PR 리뷰 자동화
```
/review-pr [PR번호]
```
- 변경사항 분석
- 리뷰 코멘트 자동 생성

### Skills 작성 Best Practices

1. **명확한 단계 정의**: skill.md에 1-2-3 단계로 작업 순서 명시
2. **예시 제공**: 실제 사용 예시를 포함하여 이해도 향상
3. **에러 처리**: 예상되는 에러 상황과 대응 방법 정의
4. **버전 관리**: skill.json에 버전 정보 기록
5. **문서화**: 주석과 설명을 충분히 작성

### Skills vs 일반 프롬프트 비교

**Skills 사용이 적합한 경우:**
- 반복적으로 수행하는 작업
- 프로젝트 규칙이 명확한 작업
- 여러 단계가 연계된 복잡한 워크플로우
- 팀 내 표준화가 필요한 작업

**일반 프롬프트가 적합한 경우:**
- 일회성 작업
- 탐색적인 작업
- 컨텍스트에 따라 크게 달라지는 작업

## 핵심 정리
- Skills는 `.claude/skills/` 디렉토리에 정의하며, skill.md와 skill.json이 필수
- 슬래시 커맨드 형태로 실행되어 반복 작업을 자동화
- 단일 책임 원칙을 지키고, 명확한 인터페이스 설계가 중요
- 프로젝트 컨벤션과 구조를 반영하여 실용성 확보
- TIL 관리, Git 워크플로우, PR 리뷰 등 다양한 영역에 적용 가능
- 반복적이고 규칙이 명확한 작업에 Skills를 활용하면 생산성이 크게 향상됨
