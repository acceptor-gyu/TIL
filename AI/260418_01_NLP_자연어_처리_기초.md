# NLP(자연어 처리) 기초

## 개요
자연어 처리(Natural Language Processing, NLP)는 인간이 일상적으로 사용하는 언어를 컴퓨터가 이해하고 처리할 수 있도록 하는 AI 분야이다. 검색 엔진, 번역기, 챗봇, 음성 비서, 스팸 필터 등 우리가 접하는 대부분의 지능형 서비스의 핵심 기술로 자리잡고 있다.

## 상세 내용

### 1. NLP란 무엇인가

NLP는 컴퓨터가 인간의 언어를 이해하고 생성할 수 있도록 하는 기술 분야다. 크게 두 가지 방향으로 나뉜다.

- **NLU (Natural Language Understanding)**: 자연어를 입력으로 받아 의미를 이해하는 방향. 텍스트 분류, 감성 분석, 질의응답 등이 여기에 해당한다.
- **NLG (Natural Language Generation)**: 의미나 구조를 입력으로 받아 자연어 텍스트를 생성하는 방향. 요약, 번역, 챗봇 응답 생성 등이 해당한다.

자연어 처리가 어려운 이유는 아래와 같다.

- **모호성(Ambiguity)**: "배"가 과일인지, 신체 부위인지, 탈 것인지 문맥 없이는 판단 불가능
- **문맥 의존성(Context Dependency)**: 같은 단어도 앞뒤 문장에 따라 의미가 달라짐
- **다의성(Polysemy)**: 하나의 단어가 여러 의미를 가짐
- **신조어 및 비정형 데이터**: 인터넷 언어, 은어, 오탈자 등

### 2. NLP의 핵심 처리 단계

실제 NLP 파이프라인은 여러 처리 단계를 거친다.

#### 텍스트 전처리

원시 텍스트를 모델이 처리할 수 있는 형태로 가공하는 과정이다.

- **토크나이제이션(Tokenization)**: 문장을 단어, 서브워드, 형태소 단위로 분리. 현대 NLP에서는 WordPiece, SentencePiece 같은 서브워드 토크나이저가 주로 사용되며 어휘 크기를 최대 40%까지 줄일 수 있다.
- **정규화(Normalization)**: 대소문자 통일, 특수문자 제거, 불용어(Stopword) 제거
- **클리닝(Cleaning)**: HTML 태그 제거, 중복 공백 처리 등

#### 형태소 분석과 품사 태깅

- **형태소 분석(Morphological Analysis)**: 어절을 의미의 최소 단위인 형태소로 분해. 한국어처럼 교착어에서는 특히 중요하다.
- **품사 태깅(POS Tagging)**: 각 토큰에 명사(NN), 동사(VB), 형용사(JJ) 등의 품사 정보를 부착

#### 구문 분석과 의미 분석

- **구문 분석(Parsing)**: 문장의 문법 구조(주어-목적어-서술어 관계)를 트리 형태로 파악
- **의미 분석(Semantic Analysis)**: 단어와 문장의 실제 의미 파악, 동의어/반의어 관계 처리
- **담화 분석(Discourse Analysis)**: 문장 간 관계, 대명사 지시 대상(Coreference Resolution) 파악

### 3. 텍스트 표현 방법

컴퓨터는 텍스트를 직접 처리할 수 없기 때문에 수치 벡터로 변환해야 한다. 이 방법은 아래처럼 발전해왔다.

#### Bag of Words (BoW)

문서를 단어의 출현 빈도 벡터로 표현한다. 단순하고 빠르지만 단어 순서와 문맥 정보를 전혀 반영하지 못한다.

```
"I love NLP" → {I: 1, love: 1, NLP: 1, ...}
```

#### TF-IDF (Term Frequency - Inverse Document Frequency)

단어 빈도(TF)와 역문서 빈도(IDF)를 곱해 단어의 중요도를 측정한다. 모든 문서에서 자주 등장하는 단어(the, is 등)의 가중치를 낮추고, 특정 문서에서만 자주 등장하는 단어에 높은 가중치를 부여한다.

```
TF(t, d)  = (문서 d에서 단어 t의 등장 횟수) / (문서 d의 총 단어 수)
IDF(t)    = log(총 문서 수 / 단어 t가 포함된 문서 수)
TF-IDF    = TF × IDF
```

#### Word Embedding

단어를 고밀도(dense) 벡터 공간에 표현하여 의미적 유사도를 포착한다. 비슷한 의미의 단어는 벡터 공간에서 가까이 위치한다.

| 방법 | 특징 |
|------|------|
| **Word2Vec** | 주변 단어를 예측(CBOW) 하거나 단어로 주변 단어를 예측(Skip-gram)하여 학습 |
| **GloVe** | 전체 코퍼스의 단어 공기(co-occurrence) 행렬을 분해해 학습 |
| **FastText** | 서브워드(n-gram) 단위로 학습해 미등록 단어(OOV) 처리에 강점 |

Word2Vec에서 유사도는 코사인 유사도로 계산한다. 예를 들어 `king - man + woman ≈ queen`이 성립하는 벡터 연산이 가능하다.

#### Contextual Embedding

기존 임베딩은 단어마다 고정 벡터를 가지므로 문맥에 따른 다의어를 처리할 수 없다. Contextual Embedding은 문장 전체 문맥을 반영하여 같은 단어라도 상황에 따라 다른 벡터를 생성한다.

- **ELMo**: 양방향 LSTM 기반, 문맥에 따라 다른 임베딩 생성
- **BERT**: Transformer 인코더 기반, 양방향 문맥 이해
- **GPT 계열**: Transformer 디코더 기반, 텍스트 생성에 특화

### 4. NLP 주요 태스크

#### 텍스트 분류 (Text Classification)
문서나 문장을 사전 정의된 카테고리로 분류한다. 스팸 분류, 뉴스 카테고리 분류, 의도(Intent) 분류 등이 해당한다.

#### 개체명 인식 (Named Entity Recognition, NER)
텍스트에서 인명(Person), 지명(Location), 조직명(Organization), 날짜(Date) 등을 자동으로 식별한다.
```
"삼성전자가 서울에서 발표회를 열었다"
→ [삼성전자: ORG] [서울: LOC]
```

#### 감성 분석 (Sentiment Analysis)
텍스트에서 긍정/부정/중립 감정을 분류한다. 리뷰 분석, 여론 조사 등에 활용된다.

#### 기계 번역 (Machine Translation)
한 언어를 다른 언어로 자동 번역한다. 현재는 Transformer 기반 신경망 번역(Neural Machine Translation)이 표준이다.

#### 질의응답 (Question Answering)
주어진 지문에서 질문에 대한 답변을 추출하거나 생성한다. BERT 계열 모델이 SQuAD 같은 벤치마크에서 인간 수준을 달성했다.

#### 요약 (Summarization)
긴 문서를 짧게 요약한다. 추출 요약(Extractive)은 원문 문장을 선택하고, 생성 요약(Abstractive)은 새 문장을 생성한다.

### 5. NLP 모델의 발전

#### 통계 기반 NLP (Statistical NLP)
- **n-gram**: 연속된 n개 단어의 등장 확률로 다음 단어를 예측
- **HMM (Hidden Markov Model)**: 시퀀스 데이터의 상태 전이 확률로 품사 태깅, 음성 인식에 활용

#### 딥러닝 기반 NLP

| 모델 | 특징 | 한계 |
|------|------|------|
| **RNN** | 순차적으로 입력을 처리, 이전 상태를 다음 스텝에 전달 | 장거리 의존성(Long-range Dependency) 학습 어려움 |
| **LSTM** | 셀 상태(Cell State)와 게이트로 장기 의존성 보완 | 병렬 처리 불가, 느린 학습 속도 |
| **GRU** | LSTM의 경량화 버전, 파라미터 감소 | LSTM과 유사한 한계 |

#### Transformer 아키텍처

2017년 "Attention is All You Need" 논문에서 제안된 아키텍처로, Self-Attention 메커니즘을 통해 입력 시퀀스의 모든 위치를 병렬로 처리한다. 현재 2025년 기준 프로덕션 NLP 모델의 90% 이상이 Transformer 기반이다.

**Self-Attention 핵심 개념:**
- Query(Q), Key(K), Value(V) 벡터를 계산
- Attention Score = softmax(QK^T / sqrt(d_k)) × V
- 문장 내 모든 단어 간 관계를 동시에 계산

**Multi-Head Attention:**
여러 개의 Attention Head를 병렬로 사용해 다양한 관계 패턴을 포착한다.

#### Pre-trained Language Model

대규모 코퍼스로 사전 학습(Pre-training) 후 특정 태스크에 파인튜닝(Fine-tuning)하는 방식이다.

| 모델 | 방향 | 강점 |
|------|------|------|
| **BERT** | 양방향(Encoder) | NLU 태스크 (분류, NER, QA) |
| **GPT** | 단방향(Decoder) | NLG 태스크 (텍스트 생성) |
| **T5, BART** | 인코더-디코더 | 번역, 요약 |

BERT-BASE는 12개의 Transformer 레이어를 갖고, BERT-LARGE는 24개를 갖는다.

#### LLM과 In-Context Learning

GPT-3 이후 모델 크기가 급격히 커지면서 Zero-shot, Few-shot 학습이 가능해졌다. 파인튜닝 없이 프롬프트만으로 다양한 태스크를 수행할 수 있는 In-Context Learning이 등장했다. 2020년 이후 최고 성능 NLP 모델의 크기는 18개월마다 10배씩 증가하고 있다.

### 6. 한국어 NLP의 특수성

한국어는 교착어(Agglutinative Language)로 어근에 다양한 어미와 조사가 결합되는 구조를 가진다. 이 때문에 영어와는 다른 전처리 전략이 필요하다.

**교착어 특성의 예:**
```
"먹다" → 먹어, 먹었다, 먹겠습니다, 먹히다, 먹이다 ...
"나는" = 나(대명사) + 는(주격 조사)
```

#### 한국어 토크나이저

KoNLPy는 파이썬 기반 한국어 NLP 라이브러리로 아래 형태소 분석기를 제공한다.

| 분석기 | 특징 |
|--------|------|
| **Mecab** | 가장 빠른 속도, 정확도 높음, 실무에서 가장 많이 사용 |
| **Okt (Open Korean Text)** | 트위터 오픈소스 기반, 신조어에 비교적 강함 |
| **Komoran** | Java 기반, 사용자 사전 추가 용이 |
| **Khaiii** | Kakao 개발, 딥러닝 기반 형태소 분석 |
| **한나눔** | KAIST 개발, 구문 분석까지 지원 |

```python
from konlpy.tag import Mecab

mecab = Mecab()
mecab.morphs("아버지가방에들어가신다")
# ['아버지', '가', '방', '에', '들어가', '신다']
```

#### 한국어 사전 학습 모델

| 모델 | 개발 | 기반 |
|------|------|------|
| **KoBERT** | SKT | BERT, 한국어 위키/뉴스 학습 |
| **KoGPT** | Kakao | GPT-2/3, 한국어 텍스트 생성 |
| **KLUE** | 업스테이지 등 공동 | 한국어 NLU 벤치마크 + 모델 |
| **HyperCLOVA** | NAVER | LLM, 8,200억 한국어 토큰 학습 |

### 7. 실무에서의 NLP 활용

#### 검색 엔진에서의 NLP

전통적인 키워드 검색은 BM25 같은 TF-IDF 기반 알고리즘을 사용한다. 최근에는 임베딩 기반 시맨틱 검색(Semantic Search)을 추가한 하이브리드 검색이 표준으로 자리잡았다.

- **BM25 (키워드 검색)**: 정확한 단어 매칭. "financial earnings" 검색 시 해당 단어가 포함된 문서만 반환
- **Dense Embedding 검색**: "financial earnings" 검색 시 "quarterly revenue"가 포함된 문서도 반환 (의미적 유사도 포착)
- **하이브리드 검색**: BM25의 정밀도와 시맨틱 검색의 의미 이해를 결합

#### RAG (Retrieval-Augmented Generation)

외부 지식을 실시간으로 검색해 LLM의 답변에 활용하는 방식이다. LLM의 환각(Hallucination) 문제를 줄이고 최신 정보를 반영할 수 있다.

```
[사용자 질문]
      ↓
[Query Embedding] → [벡터 DB 검색] → [관련 문서 청크 검색]
      ↓
[LLM에 문서 + 질문 입력] → [근거 기반 답변 생성]
```

2025년 기준 RAG는 단순 검색-생성 파이프라인을 넘어 Self-RAG, Corrective RAG(CRAG)처럼 모델이 자신의 검색 결과를 스스로 평가하고 수정하는 방향으로 발전하고 있다.

**임베딩 캐시 전략:** 자주 접근하는 문서의 벡터 표현을 캐시해두면 p95 응답 시간을 2.1초에서 450ms로 단축할 수 있다.

#### 프로덕트에 NLP 적용 시 고려사항

| 항목 | 고려 내용 |
|------|-----------|
| **비용** | 모델 크기, API 호출 횟수, 임베딩 생성 비용 |
| **지연시간** | 실시간 응답이 필요한 경우 경량 모델 or 캐싱 전략 |
| **품질** | 태스크에 맞는 모델 선택, 파인튜닝 여부 |
| **유지보수** | 모델 버전 관리, 데이터 드리프트 모니터링 |
| **언어 특수성** | 한국어 서비스라면 한국어 특화 전처리와 모델 고려 |

## 핵심 정리
- NLP는 인간의 언어를 컴퓨터가 이해/생성하도록 하는 기술이며, 전처리부터 의미 분석까지 여러 단계로 이루어진다
- 텍스트를 수치화하는 방식은 BoW/TF-IDF 같은 통계 방식에서 Word2Vec을 거쳐 BERT/GPT의 문맥 기반 임베딩으로 발전했다
- Transformer 아키텍처의 등장으로 NLP는 대규모 언어 모델(LLM) 시대로 접어들었으며, Self-Attention이 핵심 메커니즘이다
- 2025년 기준 프로덕션 NLP 모델의 90% 이상이 Transformer 기반이며, RNN 계열은 10% 미만으로 줄었다
- 한국어는 교착어 특성상 형태소 분석이 중요하며, 실무에서는 Mecab이 속도와 정확도 면에서 가장 많이 사용된다
- 실무에서는 단순히 모델 성능뿐 아니라 비용, 지연시간, 유지보수 측면을 함께 고려해야 한다
- RAG는 LLM의 환각 문제를 줄이고 최신 지식을 활용하는 핵심 아키텍처로, 2025년 현재 Agentic RAG로 발전 중이다

## 키워드
- **Tokenization**: 문장을 의미 단위(단어, 서브워드, 형태소 등)로 분리하는 전처리 과정. WordPiece, SentencePiece 같은 서브워드 방식이 현대 NLP의 표준으로, 어휘 크기를 최대 40%까지 줄일 수 있다.
- **Embedding**: 단어나 문장을 고정 차원의 벡터 공간에 표현하여 의미를 수치화하는 기법. 비슷한 의미의 단어는 벡터 공간에서 가까이 위치한다.
- **TF-IDF**: 단어 빈도(TF)와 역문서 빈도(IDF)를 곱해 문서 내 단어의 중요도를 측정하는 통계적 방법. 모든 문서에 자주 등장하는 단어의 가중치를 자동으로 낮춘다.
- **Transformer**: "Attention is All You Need"(2017)에서 제안된 Self-Attention 메커니즘 기반 아키텍처. RNN 없이 병렬 처리가 가능하며 현재 NLP의 사실상 표준 아키텍처다.
- **BERT**: Google이 개발한 양방향 Transformer 인코더 기반 사전 학습 모델. 문맥을 양방향으로 이해하며 NLU 태스크(분류, NER, QA)에 특화되어 있다. BERT-BASE는 12개, BERT-LARGE는 24개의 레이어를 갖는다.
- **GPT**: OpenAI가 개발한 디코더 기반 사전 학습 모델. 왼쪽에서 오른쪽으로의 단방향 생성에 특화되어 있으며 ChatGPT의 기반 모델이다.
- **NER(Named Entity Recognition)**: 텍스트에서 인명, 지명, 조직명, 날짜 등 특정 개체명을 자동으로 식별하는 NLP 태스크.
- **POS Tagging**: 각 단어에 명사, 동사, 형용사 등 품사 정보를 부착하는 형태소 분석 단계. 한국어에서는 조사와 어미 분리가 핵심이다.
- **RAG (Retrieval-Augmented Generation)**: 외부 지식을 실시간으로 검색하여 LLM의 답변 생성에 활용하는 아키텍처. LLM의 환각 문제를 줄이고 최신 정보를 반영할 수 있다. 2025년 현재 Self-RAG, Agentic RAG로 발전 중이다.
- **형태소 분석**: 한국어처럼 교착어에서 어절을 의미의 최소 단위(형태소)로 분해하는 과정. 실무에서는 KoNLPy의 Mecab이 속도와 정확도 면에서 가장 많이 활용된다.
